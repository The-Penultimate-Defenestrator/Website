More Computer Science-oriented Tools on School Computers

Hi, I'm Luke Deen Taylor. I'm a technologically inclined New Paltz High School freshman, I'll be taking AP Computer Science next year. On behalf of myself and a few other students, I'd like to discuss the possibility of installing more computer science-oriented tools on school computers. Specifically, I'd love to see the following things available to students:

  1. Installation of an IDE besides Notepad++

    Many high school computers have Notepad++ as a code editor installed. While Notepad++ is fine, it's a relatively antiquated text editor, lacking many productivity features found commonly in more modern text editors. Editors like Atom feature code-completion, advanced syntax highlighting, and more out of the box. Atom also has linters available for languages such as JavaScript, which can help prevent mistakes in writing code by identifying them before execution time. I'd love to see Atom installed system-wide in coming years.

  2. Installation of other browsers besides Internet Explorer

    Keith in the IT department may remember being called by Dr. Weisenthal last year about installing Chrome on school computers. From what I can see, a halfhearted attempt was subsequently made to block its installation by blacklisting the chrome.google.com domain. However, Internet Explorer remains an awful, non-standards-compliant browser. It doesn't support many of the features of the modern web. Browsers such as Chrome not only offer a full, standards-compliant implementation of HTML, CSS and JavaScript, but also offer debugging tools far beyond the capabilities of those found in IE. Having Google Chrome installed on school computers would be very helpful (it's not evil).

  3. Relaxation of computer AUP

    The district has a computer acceptable use policy, designed to protect the security of district computers. However, this policy is, in my opinion, unnecessarily restrictive in several places.

    The computer use policy has not been modified or updated in several years to reflect how students need to use technology today. At least two of these rules are worded such that they are violated by normal, reasonable computer use. For example, launching Internet Explorer and conducting a Google search explicitly violates the rules stating that users may not: 

    25. Run or copy executable programs for any drive on any of the district's computers
27. Send messages from one computer to another via the LAN or WAN
 9. Complete and/or submit forms found on websites without permission
    The act of launching internet explorer involves running an executable program for a drive on a district computer. Furthermore, navigating to any webpage on the internet, by definition, involves sending a message from one computer to another over "the WAN." Lastly, conducting a search involves completing and submitting a form found on a website. In fact, Google's input mechanism explicitly uses a HTML <form> element:

    The district's computer policies cannot be expected to be taken seriously when they clearly prevent normal computer use.

    In addition to the clear mistakes in the policy, there are several rules that I feel are overly restrictive. For example, rule 24 states: 

    24. Load software or executable files of any kind onto any of the district's computers or network server
    This rule, is, in my opinion, unnecessary. Without access to adminstrator privelages, no programs could be loaded which would inflict actual damage to the computers or the network, at least not without violating the rule: 
    3. Create and/or place a computer virus onto any computer
    I think the computer AUP should undergo a much-needed update sometime very soon.

  4. Basic, non-admin command line access

    While Windows is built on top of a graphical interface, familiarity with a basic command-line interface is an essential tool for students hoping to become developers. Currently, placing cmd in a .bat file and double-clicking it displays the message: 

    The command prompt has been disabled by your administrator.
Press any key to continue . . .
    this restriction can be easily circumvented by a loop in the batch file: 
    @echo off
:loop
set /P cmd="> "
%cmd%
goto :loop
    which simply passes user input to the command line, simulating an interactive shell: Allowing non-admin command-line access would be hugely beneficial to students while not actually opening up any new security issues. This addition would not expose new functionality, but rather make information (which is already available) accessible in a much more powerful way.

  5. Installation of Git

    While many computer science students would not take advantage of a version-control system such as git, since it is not a part of the curriculum, such software has enormous potential to transform code workflows.

I'd love to discuss these posibilities. Are there reasons why any of these could or could not be put in place? I'm open to feedback on these ideas.
